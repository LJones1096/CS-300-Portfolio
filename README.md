Reflection

What was the problem you were solving in the projects for this course?
The main problem I was solving in this course was how to store, organize, and retrieve course data efficiently. In Project One, I focused on analyzing different data structures and how they perform in terms of runtime and memory. In Project Two, I had to actually implement a system that could load course data, sort it, and display it in alphanumeric order.

How did you approach the problem?
I approached the problem by breaking it down into smaller steps. First, I focused on understanding how different data structures like vectors, hash tables, and binary search trees work. Then I compared them to figure out which one would be the best fit. Data structures are important because they directly impact how fast and efficient a program runs, especially when dealing with larger amounts of data.

How did you overcome any roadblocks?
Some of the biggest challenges were understanding how to properly organize the data and making sure everything worked correctly when loading and sorting the courses. I got through it by testing small parts of the code at a time and fixing errors as they came up. If something didn’t work, I would go back, check my logic, and adjust it until it worked correctly.

How has this project expanded your approach to designing software?
This project helped me realize how important planning is before writing code. Instead of just jumping in, I now think more about which data structure to use and why. It also showed me how different approaches can impact performance, which is something I didn’t pay as much attention to before.

How has this project changed the way you write programs?
I now focus more on making my code readable and organized. I try to write code that is easier to understand and modify later instead of just making it work. I also think more about how my code can be reused or expanded in the future.
